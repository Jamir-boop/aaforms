AA Forms — Automation Recorder Training

Local test site for learning and testing web automation recorder tools

Objective

This site exists for learning to use automation recorder tools (such as Automation Anywhere's web recorder) and for testing how they behave against real-world pages that are deliberately difficult to automate.

The plain form.html is a normal SAP-style form — the easy case. Each case-*.html page below applies one technique (or all of them) that commonly breaks recorders, so you can practice handling them:

  • srcdoc + sandbox — iframe with no URL to navigate to
  • runtime-built DOM — fields created by JS after load
  • randomized IDs — change on every load
  • vision fallback — 1800 off-screen decoys push the target past the labeled-DOM size limit, forcing screenshot-based selection
  • shadow DOM — fields hidden behind a shadow boundary
  • contenteditable divs — instead of <input>
  • no IDs, addEventListener only — no stable selector anchors
  • div submit button — a <div> acting as the submit button
  • nested iframe — a frame inside a frame
  • all techniques stacked — every technique above combined

Open the published GitHub Pages site and record against each section. Observe where the recorder fails, then figure out the workaround — that is the point of this site.
